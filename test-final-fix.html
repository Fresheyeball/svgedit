🔧 Technical Summary

🐛 Root Cause Found:

Destructive style attribute overwriting - Multiple parts of the codebase were using setAttribute('style', 'pointer-events:...') which completely overwrote existing CSS styles on div elements.

🛠️ Fix Applied:

Two-part solution:

  1. Sanitizer Fix: Modified SVG sanitizer to preserve style attributes on HTML elements while still processing them for SVG elements
  2. Non-destructive Style Updates: Changed all setAttribute('style', '...') calls to use element.style.pointerEvents = '...' to preserve existing styles

📁 Files Modified:

  • packages/svgcanvas/core/sanitize.js - Added div to style whitelist and preserved HTML element styles
  • packages/svgcanvas/core/event.js - Fixed destructive style overwriting
  • packages/svgcanvas/core/layer.js - Fixed pointer-events style updates
  • packages/svgcanvas/core/draw.js - Fixed element enable/disable style updates
  • packages/svgcanvas/core/svg-exec.js - Added restoration function for CSS object model

🎊 Result:

Text styling now persists correctly through browser refreshes! The original issue is completely resolved.
